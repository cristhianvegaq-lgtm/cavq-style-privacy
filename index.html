CAVQ_Style es un asistente de reescritura científica. El responsable indicado determina las finalidades y condiciones de la base externa de mejora denominada CAVQ_Style Metrics.

2. Alcance

Esta política regula exclusivamente la información que una persona autorice enviar mediante la Action de CAVQ_Style a la API externa CAVQ_Style Metrics.

  • Política pública: https://cristhianvegaq-lgtm.github.io/cavq-style-privacy/
  • API externa: https://cavq-style-metrics-api.onrender.com

El uso general de ChatGPT y el contenido que permanezca dentro de ChatGPT se rigen por los términos y políticas de OpenAI. El responsable de CAVQ_Style Metrics no tiene acceso general a las conversaciones completas mantenidas por los usuarios.

3. Finalidades

  1. Evaluar la calidad de la reescritura científica.
  2. Comparar el desempeño entre campos, diseños y secciones de investigación.
  3. Identificar fallos lingüísticos, metodológicos, disciplinares e inferenciales.
  4. Mejorar las instrucciones, protocolos, ejemplos y corpus de CAVQ_Style.
  5. Evaluar nuevas versiones mediante conjuntos de desarrollo, validación y prueba.
  6. Elaborar informes estadísticos agregados.

La información no se utilizará para publicidad, venta de datos, elaboración de perfiles personales, identificación de autores ni decisiones automatizadas sobre usuarios.

4. Participación voluntaria y niveles de contribución

La reescritura funciona aunque el usuario no autorice contribución alguna. No existe una opción seleccionada por defecto.

A. Sin contribución

No se envía información a CAVQ_Style Metrics.

B. Métricas anónimas

Podrán registrarse categorías científicas, métricas lingüísticas, diferencias entre versiones, controles de calidad, etiquetas cerradas de error, versión del GPT, rondas y valoración categórica. No se almacenan textos, archivos, nombres, correos ni identificadores de cuenta o conversación.

C. Métricas y textos anonimizados

Podrán conservarse el fragmento de entrada, la primera reescritura y la versión aceptada después de anonimización. El usuario deberá confirmar que tiene derecho a compartir el material y que no contiene información personal o sensible identificable.

El silencio, la falta de respuesta o una respuesta ambigua equivalen a ausencia de autorización.

5. Información excluida

No se pretende recopilar nombres, correos, identificadores de usuarios o conversaciones, documentos de identidad, direcciones personales de terceros, historias clínicas, datos genéticos o biométricos, imágenes clínicas identificables, información identificable de menores, información sensible identificable ni secretos no autorizados.

Cuando exista una posibilidad razonable de identificación, el texto no se enviará. Con autorización podrán registrarse únicamente métricas anónimas.

6. Proveedores técnicos y ubicación

Render

  • Función: alojamiento de la API y PostgreSQL.
  • Región configurada: Virginia, Estados Unidos.
  • API: https://cavq-style-metrics-api.onrender.com

GitHub Pages

  • Función: publicación pública de esta política.
  • Sitio: https://cristhianvegaq-lgtm.github.io/cavq-style-privacy/

El uso de proveedores ubicados o administrados fuera de Ecuador puede implicar acceso, alojamiento o tratamiento técnico desde el extranjero.

7. Conservación

  • Métricas individuales: hasta 36 meses.
  • Textos anonimizados: hasta 24 meses.
  • Copias de seguridad: hasta 90 días.
  • Informes agregados: durante la vigencia del proyecto cuando no permitan individualizar contribuciones.

8. Acceso y seguridad

El acceso estará limitado al rol Administrador. Se aplicarán autenticación segura, HTTPS/TLS, separación entre métricas y textos, cifrado de textos anonimizados, control de acceso, copias de seguridad, registros de operación, minimización y pruebas de eliminación.

El entorno gratuito de Render se utilizará únicamente para pruebas técnicas. La recopilación estable requerirá infraestructura persistente, respaldo y controles adecuados.

9. Validación y categorías de error

La valoración se almacenará mediante categorías cerradas: aceptada, aceptada con cambios menores, requiere cambios sustantivos, rechazada o sin respuesta. No se almacenarán comentarios libres salvo consentimiento ampliado y anonimización.

10. Eliminación de contribuciones

Cada contribución autorizada generará un código de eliminación. Para solicitar la supresión, el usuario deberá enviar únicamente dicho código a cristhian.vega.q@gmail.com.

La eliminación comprenderá métricas, textos anonimizados, validación, etiquetas y registro activo. Los resultados agregados de forma irreversible podrían no ser reversibles individualmente.

11. Derechos

El usuario podrá solicitar información, retirar el consentimiento y pedir eliminación mediante el correo o el teléfono de contacto.

12. Cambios

La política se actualizará cuando cambien los datos, finalidades, plazos, proveedores, región, consentimiento, seguridad o funcionamiento de la Action.
